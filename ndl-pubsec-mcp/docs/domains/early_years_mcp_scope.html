Purpose. This document scopes an MCP server enabling AI assistants to help parents, carers, and early years professionals navigate support for young children's development — from understanding developmental milestones and accessing speech therapy, to navigating SEND assessments and finding family hubs. It aligns with the NDL Early Years Kickstarter (announced 4 June 2026) and the government's goal of ensuring every child gets the best start in life. Pilots running in Leeds, London Borough of Hammersmith & Fulham, and Liverpool City Region.

Problem statement

Health visiting, early education, and childcare operate in silos with paper-based processes. A health visitor might notice a developmental concern. An early years practitioner might observe the same child struggling. But if those observations sit on separate pieces of paper in separate systems, no one joins the dots — and the child falls through the gap.

The burden of navigating this fragmentation falls on parents — often at their most vulnerable, sleep-deprived, and anxious. The parents least equipped to navigate complexity are disproportionately those whose children face the greatest developmental risk: low-income families, single parents, families where English is not the first language, and families in areas with depleted early help infrastructure.

The NDL Early Years Kickstarter explores how to connect health visiting, education and childcare provider data. This MCP addresses the citizen-facing layer: helping parents understand what support exists, how to access it, and what their and their child's rights are.

MCP tool specifications

This MCP serves two distinct user types — parents/carers and early years professionals — and the tools must be calibrated accordingly. Tools serving parents should never make parents feel their child is being assessed or judged. Tools serving professionals must maintain clear boundaries around what constitutes a safeguarding concern vs. a developmental referral.

eyfs_check_developmentBuild first
Returns age-appropriate developmental milestones with clear, compassionate guidance on when to seek support. Uses NHS and DfE EYFS guidance. Not a diagnostic tool — helps parents understand the difference between "typical range" and "worth talking to a professional about." Foundational to all other tools.
interface EyfsCheckDevelopmentInput { child_age_months: number; // age in months — developmental stage more useful than years concern_area?: string[]; // "speech"|"motor"|"social"|"play"|"attention"|"sleep" reading_level?: 'simple' | 'standard'; language?: string; user_type?: 'parent' | 'professional'; // affects tone and depth } interface EyfsCheckDevelopmentOutput { milestones: Milestone[]; // typical range for this age in the concern area in_typical_range: boolean; // based on described concerns — NOT a diagnosis when_to_seek_help: string; reassuring_context: string; // children develop at different rates; tone calibrated next_steps: string[]; disclaimer: string; // "This is not a diagnosis. Contact your health visitor or GP if concerned." }
eyfs_find_support
Finds local support services by postcode — Best Start Family Hubs, speech and language therapy self-referral pathways, health visitor contacts, portage services, and children's centres. The most practically impactful tool for a parent who knows something is wrong but doesn't know where to go.
interface EyfsFindSupportInput { postcode: string; support_type?: SupportType[]; // "speech_language"|"family_hub"|"portage"|"parenting"|"health_visitor" child_age_months?: number; language?: string; } interface EarlyYearsService { name: string; type: SupportType; description: string; address?: string; contact: ContactInfo; self_referral: boolean; // can parent refer directly, or needs professional referral? wait_time_est?: string; cost: 'free' | 'means_tested' | 'paid'; }
eyfs_navigate_send
Step-by-step guide to the SEND (Special Educational Needs and Disabilities) system for young children — from initial concern to EHCP. Explains the difference between SEND Support (school/setting action) and an EHCP. Covers the SEND Code of Practice rights and timelines.
interface EyfsNavigateSendInput { stage: 'just_concerned' | 'referred' | 'assessment' | 'send_support' | 'ehcp_request'; child_age_months: number; setting_type?: 'home' | 'nursery' | 'childminder' | 'school'; local_area?: string; // postcode — SEND local offer varies by council language?: string; } interface EyfsNavigateSendOutput { current_stage_explained: string; next_steps: SENDStep[]; parents_rights: string[]; // SEND Code of Practice rights timelines: Timeline[]; local_send_offer_url?: string; parent_carer_forum: Organisation; }
eyfs_explain_entitlements
Explains what parents and children are entitled to — 15/30 hours funded childcare, Healthy Start vouchers, Early Help assessment, Pupil Premium for disadvantaged 3 and 4-year-olds, and Disability Living Allowance for children. Many entitlements are unclaimed.
interface EyfsExplainEntitlementsInput { entitlement_name?: string; // specific entitlement, or omit for full overview child_age_months: number; family_context?: FamilyContext; // work status, income band, housing — affects eligibility reading_level?: 'simple' | 'standard'; language?: string; } interface EyfsExplainEntitlementsOutput { entitlements: Entitlement[]; gaps_identified: string[]; // entitlements not currently being used application_steps: ApplicationStep[]; further_info_url: string; }
eyfs_find_professional_guidance
For early years professionals (childminders, nursery workers, health visitors): guidance on professional reporting obligations, how to raise developmental concerns with parents constructively, safeguarding referral pathways, and the Prevent duty in early years settings.
interface EyfsFindProfessionalGuidanceInput { professional_role: string; // "childminder"|"nursery_worker"|"health_visitor"|"SENCO" guidance_type: string; // "reporting_concern"|"talking_to_parents"|"safeguarding"|"SEND_referral" setting_type?: string; } interface EyfsFindProfessionalGuidanceOutput { obligations: string[]; // legal and regulatory obligations guidance_steps: string[]; how_to_approach: string; // how to raise concerns with parents sensitively referral_pathways: ReferralPath[]; regulatory_sources: string[]; // EYFS statutory framework, Keeping Children Safe etc. }
eyfs_connect_parents
Finds parenting courses, SEND parent-carer forums, peer support groups, and family learning opportunities by postcode. Social connection and peer support are themselves protective factors for children's development — this tool serves the parent's wellbeing as well as the child's.
interface EyfsConnectParentsInput { postcode: string; support_type?: 'parenting_course' | 'send_forum' | 'peer_support' | 'family_learning' | 'all'; language?: string; child_age_months?: number; } interface EyfsConnectParentsOutput { groups: ParentGroup[]; online_options: OnlineResource[]; send_parent_forum: Organisation; // always returned — critical for SEND navigating families nct_contact?: ContactInfo; }

Data architecture

DfE EYFS statutory framework

GOV.UK, free, public. The statutory framework all early years settings must follow — primary source for professional guidance and developmental expectations.

NHS child development milestones

NHS.uk public guidance on developmental milestones by age. Clear, evidence-based. Core to eyfs_check_development — authoritative and already in accessible language.

SEND Code of Practice

Statutory guidance (GOV.UK). Covers rights, timelines, and obligations for SEND support and EHCPs. Core to eyfs_navigate_send — legally authoritative.

Local authority SEND local offers

Every council must publish a SEND local offer (legal requirement). Highly variable quality. Web search required per postcode. Core to local service finding.

NDL early years linked data Phase 2+

The NDL Early Years Kickstarter connecting health visiting, education and childcare provider data. When available, enables proactive identification of children needing support — the core innovation of this kickstarter.

DEIA requirements

Multi-language — FSM-eligible families disproportionately include EAL households; Welsh, Urdu, Somali, Romanian, Arabic as minimum
Never diagnostic — explicitly clear that tools support navigation, not assessment of children
Parent wellbeing aware — tone that doesn't increase parental anxiety unnecessarily
WCAG 2.2 AA throughout — accessible to parents with their own disabilities
Mobile-first — new parents access information on phones, often one-handed
No login for milestones and entitlement information — consent-first for personal data
Culturally sensitive — attitudes to child development, disability and professional support vary
Professional/parent mode — content calibrated for the user type from the start
Plain English — EHCP, SEND, EHNA, portage all explained; FK Grade 6 for parent-facing
GDS service standard compliant — built around real parent/professional journeys
Safeguarding aware — professional tools know where referral vs. support is appropriate
Co-designed with families — SEND parent-carer forums, health visitors, childminders from alpha

Recommended build order

  1. 1

    eyfs_check_development

    Simplest to build. NHS milestone guidance from GOV.UK/NHS.uk — public, authoritative, already in accessible language. The tool that most parents need first and that builds trust for the rest of the journey. Critical: tone must be non-anxious, non-diagnostic.

  2. 2

    eyfs_find_support

    Converts milestone awareness into action. Best Start Family Hubs locator plus SALT self-referral pathways. Web search for local services. High immediate impact for families in the three NDL pilot areas (Leeds, Hammersmith, Liverpool).

  3. 3

    eyfs_explain_entitlements

    GOV.UK Content API for 15/30 hours childcare, Healthy Start, Pupil Premium. Many entitlements go unclaimed — this is a gap with significant financial value to families.

  4. 4

    eyfs_navigate_send

    Requires encoding SEND Code of Practice timelines and local offer data. High complexity but highest impact for families currently in the SEND system or waiting for assessment.

  5. 5

    eyfs_find_professional_guidance

    Builds on statutory framework and safeguarding guidance. Serves Gloria and the 300,000+ early years professionals in England. Different user type with different liability considerations.

  6. 6

    eyfs_connect_parents

    Requires local parenting group directory — most complex to keep current. Build last but ensure SEND parent-carer forum fallback is present from day one (every council must publish this).

Questions for the lab

  1. The NDL Early Years Kickstarter was just announced (4 June 2026). Who are the DSIT and DfE leads on this project — we should make contact immediately given the lab's positioning.
  2. The three pilot LAs (Leeds, Hammersmith & Fulham, Liverpool City Region) are starting user research now — can the lab embed alongside this research to shape the citizen-facing tool from the outset?
  3. SEND waiting lists of 12-18 months for autism assessment are a national scandal. Does this tool need to address what to do while waiting, not just how to access the process?
  4. The professional guidance tool (for childminders and nursery workers) has different liability considerations than the parent tools — should these be scoped as separate MCP servers?
  5. Co-design recommendation: SEND parent-carer forums (National Network of Parent Carer Forums), health visitors, childminders, Family Hubs Network, and IPSEA (legal rights for SEND families).
