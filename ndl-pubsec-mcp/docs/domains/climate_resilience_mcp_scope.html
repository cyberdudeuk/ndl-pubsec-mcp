Climate Resilience MCP

Scope & Architecture · NDL Kickstarter Domain 5

Draft v0.1 Cross-Gov AI Engineering Lab June 2026 For review and comment

Purpose. This document scopes an MCP server enabling AI assistants to help citizens — particularly those least able to adapt to climate risks — understand their flood, heat and cold risks, know their rights, access adaptation grants, and connect to local preparedness resources. It aligns with the NDL Kickstarter on expanding the use of weather and climate data to help organisations and individuals plan for severe weather and long-term climate change.

Problem statement

Climate risk is regressive: it falls hardest on those with the least capacity to adapt. Elderly people in flood-risk areas who haven't signed up to alerts. Outdoor workers facing extreme heat with no knowledge of their employer's legal obligations. Rural communities and small farmers facing existential agricultural risk with no access to professional climate advice.

At the same time, significant support exists — Flood Re, local authority adaptation schemes, agricultural grants, EA alerts — that goes unclaimed because the right information never reaches the right people. The NDL Kickstarter creates the data infrastructure; this MCP creates the citizen-facing access layer.

MCP tool specifications

climate_check_flood_riskBuild first
Uses the Environment Agency's free, public Flood Risk API to return a plain English explanation of flood risk for any postcode. One of the best public APIs available — returns flood zone classification, probability, and risk type (river/sea/surface water). Foundational to all flood-related tools.
interface ClimateCheckFloodRiskInput { postcode: string; include_future?: boolean; // include projected 2050/2080 risk under climate scenarios reading_level?: 'simple' | 'standard'; language?: string; } interface ClimateCheckFloodRiskOutput { risk_level: 'high' | 'medium' | 'low' | 'very_low'; risk_type: string[]; // ["river","surface_water","sea"] plain_explanation: string; future_risk?: FutureRisk; // 2050/2080 projections if requested probability_1_in: number; // e.g. 100 = 1-in-100-year flood recommended_actions: string[]; ea_alert_url: string; // link to sign up for EA flood alerts for this area }
climate_find_flood_support
Finds financial and practical support for flood risk — Flood Re (subsidised insurance), council property adaptation schemes, EA-funded flood defences, and emergency support if flooding has occurred. Connects the risk information to actionable help.
interface ClimateFindFloodSupportInput { postcode: string; situation: 'at_risk' | 'flooded_now' | 'recently_flooded'; property_type?: 'owned' | 'rented' | 'social_housing'; } interface ClimateFindFloodSupportOutput { flood_re_eligible: boolean; local_schemes: FloodScheme[]; emergency_contacts: ContactInfo[]; // EA, council emergency team, insurance adaptation_grants: Grant[]; insurance_guidance: string; }
climate_explain_weather_rights
Returns legal rights and employer obligations for extreme weather situations — workplace temperature, outdoor working in heat, cold housing standards, and tenant rights during extreme cold. Many workers and tenants don't know these rights exist.
interface ClimateExplainWeatherRightsInput { situation_type: string; // "outdoor_heat"|"indoor_heat"|"cold_housing"|"flooding_tenant" employment_type?: string; // "employee"|"worker"|"self_employed" — affects rights sector?: string; // "construction"|"agriculture"|"logistics" — sector-specific guidance reading_level?: 'simple' | 'standard'; } interface ClimateExplainWeatherRightsOutput { rights: WeatherRight[]; employer_duties: string[]; what_you_can_do: string[]; if_rights_denied: string; regulatory_body: string; // HSE, MHCLG etc. legal_sources: string[]; }
climate_find_resilience_grants
Finds available grants for home, business, or agricultural climate adaptation by location and property type. Covers Boiler Upgrade Scheme, Energy Efficiency schemes, Countryside Stewardship, and local authority climate adaptation funds.
interface ClimateFindResilienceGrantsInput { postcode: string; applicant_type: 'homeowner' | 'private_renter' | 'business' | 'farmer'; grant_type?: 'flood' | 'energy' | 'heat' | 'agricultural' | 'all'; property_type?: string; } interface ClimateResilienceGrant { name: string; description: string; max_value: string; eligibility: string; deadline?: string; application_url: string; administering_body: string; }
climate_find_local_preparedness
Finds local emergency plans, community flood action groups, volunteer resilience networks, and how to sign up for EA flood warnings for a specific location. Connects citizens to their local preparedness infrastructure.
interface ClimateFindLocalPreparednessInput { postcode: string; risk_type?: 'flood' | 'heat' | 'cold' | 'all'; } interface ClimateFindLocalPreparednessOutput { local_resilience_forum: string; community_groups: Organisation[]; ea_alert_signup_url: string; council_emergency_plan: string; volunteer_opportunities: string[]; }
climate_assess_vulnerability
Assesses personal vulnerability to climate hazards based on age, health conditions, housing type, location and employment. Returns tailored next steps and priority actions. Bridges from general risk information to personalised action.
interface ClimateAssessVulnerabilityInput { postcode: string; age_band?: string; // "under_5"|"5_to_64"|"65_to_74"|"75_plus" health_conditions?: string[]; // affects heat/cold vulnerability scoring housing_type?: string; // "top_floor_flat"|"detached"|"semi"|"terraced" employment_type?: string; } interface ClimateAssessVulnerabilityOutput { vulnerability_profile: VulnerabilityScore[]; // by hazard type priority_risks: string[]; recommended_actions: PrioritisedAction[]; local_support: Organisation[]; }

Data architecture

EA Flood Risk API Outstanding

Free, public, excellent REST API. Returns flood zone, risk probability, risk type by postcode. Base: environment.data.gov.uk/flood-monitoring. Build first tool on this API.

Met Office APIs

Weather observation and forecast data. Some free tiers. Heat-health alert data. Useful for heat/cold risk tools and real-time severe weather guidance.

GOV.UK guidance

Heat-health alert guidance, flooding advice, HSE outdoor working guidance, agricultural climate advice. Free, public, Content API accessible.

Flood Re data

Public information on Flood Re scheme — insurance availability for high-risk properties. No API but structured guidance on eligibility and how to access.

NDL weather & climate data Phase 2+

The NDL kickstarter expands access to Met Office and other climate datasets for planning. Will enable richer, combined risk assessments linking property, demographic and climate data.

DEIA requirements

Multi-language — climate impacts on migrant communities, Welsh bilingual required
WCAG 2.2 AA — elderly users are a primary cohort; large text, simple navigation
Rural accessibility — works on poor mobile connectivity; low-bandwidth design
Plain English — no climate science jargon; "1-in-100-year flood" explained clearly
Mobile-first — emergency use case; must work during a flood event on a phone
Offline resilience — key safety information cached where possible
No login for risk information — consent-first for location-based personalisation
Alert sign-up as a core journey — EA flood alerts are the highest-value single action
Agricultural communities — guidance available in Welsh; rural internet limitations considered
GDS service standard compliant — built for real emergencies, not just informational use
BAME health impact — darker skin tone increases heat risk; guidance appropriately tailored
Co-designed with communities — local resilience forums, NFU, flood action groups from alpha

Recommended build order

  1. 1

    climate_check_flood_risk

    Best public API in the government space. Environment Agency Flood Risk API is free, well-documented, and returns rich data. Build this first — it's the gateway to all flood tools and demonstrates tangible value immediately.

  2. 2

    climate_find_flood_support

    Builds directly on the risk tool. Adding Flood Re eligibility and local scheme information converts risk awareness into action. High immediate impact.

  3. 3

    climate_explain_weather_rights

    Uses GOV.UK + HSE guidance. Outdoor heat rights are a pressing issue and currently underserved. Build after flood tools are working.

  4. 4

    climate_find_resilience_grants

    Requires curating grant data from multiple sources. High financial value for recipients. Build after core risk and rights tools.

  5. 5

    climate_find_local_preparedness

    Requires local resilience forum data — currently scattered across council websites. Web search integration needed.

  6. 6

    climate_assess_vulnerability

    Most complex — requires integrating risk, demographic, health and housing data. Build last but highest long-term value for targeted support.

Questions for the lab

  1. Is the NDL climate kickstarter primarily about Met Office data access, or is it also connecting demographic/vulnerability data to climate risk? This changes scope significantly.
  2. The EA Flood Risk API is excellent — are there equivalent quality APIs for heat or cold risk we should be aware of?
  3. Outdoor worker heat rights are legally ambiguous (no specific temperature threshold in UK law unlike some EU countries) — how do we communicate this accurately without misleading users?
  4. Agricultural climate guidance overlaps with DEFRA's Countryside Stewardship and Farming in Protected Landscapes — are there DEFRA colleagues we should collaborate with?
  5. Co-design recommendation: Environment Agency, Met Office, Flood Re, NFU, local resilience forums, and community flood action groups.
