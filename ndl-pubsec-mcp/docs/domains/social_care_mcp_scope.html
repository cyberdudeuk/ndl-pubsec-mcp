Adult Social Care MCP

Scope & Architecture · NDL Kickstarter Domain 3

Draft v0.1 Cross-Gov AI Engineering Lab June 2026 For review and comment

Purpose. This document scopes an MCP server enabling AI assistants to help people navigate adult social care — from understanding their Care Act rights and requesting a needs assessment, to exploring funding options and finding local services. It aligns with the NDL Kickstarter on adult social care data and addresses the citizen-facing layer of a system that is deeply fragmented across 150+ local authorities.

Problem statement

Adult social care is one of the most complex and least navigable parts of the UK welfare system. Data is fragmented across 152 upper-tier councils, each operating differently. Assessment waiting times in some areas exceed two years. People eligible for the Priority Services Register, direct payments, or NHS Continuing Healthcare often never claim them because they don't know they exist or don't understand their rights under the Care Act 2014.

The NDL Kickstarter targets better-connected care data to enable smarter planning. This MCP targets the person trying to understand what they or a loved one are entitled to — before, during and after a care needs assessment.

MCP tool specifications

care_explain_rightsBuild first
Returns a plain-language explanation of a person's rights under the Care Act 2014 for a given situation. This is the foundational tool — most users don't know their rights exist before they can navigate the system. Uses GOV.UK statutory guidance.
interface CareExplainRightsInput { situation: string; // "needs assessment" | "waiting too long" | "carer's assessment" | "appeal" council_area?: string; // postcode or council name for local context reading_level?: 'simple' | 'standard'; language?: string; } interface CareExplainRightsOutput { rights: Right[]; // plain English description of each right legal_basis: string; // "Care Act 2014, Section X" council_must_do: string[]; timelines: Timeline[]; // what must happen by when if_not_followed: string; // complaint and escalation path further_info_url: string; }
care_check_eligibility
Assesses likely eligibility for council-funded care based on Care Act wellbeing outcomes framework. Returns a plain English assessment and recommended next step. Not a formal assessment — helps people understand before they enter the system.
interface CareCheckEligibilityInput { needs_description: string; // free text description of needs and daily challenges household_situation: string; // living alone, with family, in residential care etc. current_support: string[]; // existing informal and formal support financial_situation?: string; // asset level if known — affects means testing } interface CareCheckEligibilityOutput { likely_eligible: boolean; eligible_domains: WellbeingDomain[]; // Care Act 2014 S.1 domains confidence_level: 'high' | 'medium' | 'low'; recommended_action: string; funding_pathway: 'council' | 'nhs_chc' | 'self_fund' | 'mixed'; }
care_navigate_assessment
Step-by-step preparation guide for requesting and participating in a care needs assessment. Helps people know what to expect, what to say, and how to ensure their needs are properly captured — particularly important for those who may minimise difficulties or struggle to articulate needs under pressure.
interface CareNavigateAssessmentInput { assessment_type: 'needs' | 'carers' | 'young_persons' | 'mental_capacity'; conditions?: string[]; council_area?: string; } interface CareNavigateAssessmentOutput { how_to_request: string; what_to_expect: AssessmentStage[]; preparation_tips: string[]; what_to_say: string[]; // example phrases for common questions documents_helpful: string[]; after_assessment: string; }
care_explore_funding
Explains the different funding pathways for adult social care — NHS Continuing Healthcare, council-funded care, direct payments, independent living funds, and self-funding — and helps navigate between them. The NHS CHC path is frequently missed and worth significant money to eligible individuals.
interface CareExploreFundingInput { care_type: string; // "residential" | "domiciliary" | "respite" | "day_care" conditions?: string[]; approximate_assets?: string; // "under_23k" | "23k_to_100k" | "over_100k" — rough banding } interface CareExploreFundingOutput { funding_options: FundingOption[]; nhs_chc_eligible: boolean; nhs_chc_pathway: string; // how to request a CHC checklist direct_payments: string; // plain English explanation if applicable financial_assessment: string; // what the means test involves }
care_find_services
Finds CQC-registered local care providers, day centres, direct payment support organisations, and respite services by postcode. Uses the CQC Provider API — one of the best public APIs in the UK public sector.
interface CareFindServicesInput { postcode: string; service_type: CareServiceType; // "domiciliary"|"residential"|"day_care"|"respite"|"supported_living" conditions?: string[]; // filters providers with relevant specialisms rating_min?: number; // CQC rating filter: 1-4 radius_km?: number; } interface CareProvider { name: string; cqc_rating: string; // "Outstanding"|"Good"|"Requires improvement"|"Inadequate" specialisms: string[]; address: string; contact: string; last_inspection: string; cqc_report_url: string; }
care_find_advocacy
Routes to the most appropriate advocacy service — Independent Mental Capacity Advocate (IMCA), Care Act advocate, Age UK, local carers centres, CQC complaints pathway. Always includes a fallback to Age UK national helpline.
interface CareFindAdvocacyInput { postcode: string; need_type: AdvocacyNeedType; // "assessment_support"|"complaint"|"carer_support"|"legal_rights" language?: string; mental_capacity?: boolean; // triggers IMCA pathway } interface AdvocacyService { name: string; type: string; // "IMCA"|"Care Act advocate"|"voluntary sector" description: string; contact_methods: ContactMethod[]; languages: string[]; is_free: boolean; }

Data architecture

GOV.UK Care Act guidance

Statutory guidance (public, free). Primary source for care_explain_rights and care_navigate_assessment — the legal framework underpinning everything.

CQC Provider API Best public API

Free, well-documented, RESTful. Returns registered providers with ratings, inspection dates, specialisms and location. Core data for care_find_services.

NHS England guidance

NHS Continuing Healthcare framework (public). Critical for care_explore_funding — the NHS CHC pathway is systematically underused by eligible individuals.

Local authority websites

Highly variable — each of 152 councils operates differently. Web search required. Used for local direct payment rates, waiting time information, and local charging policies.

NDL linked care data Phase 2+

Connected social care datasets across councils — the core of the NDL adult social care kickstarter. Enables better planning and proactive identification of unmet need.

DEIA requirements

Multi-language — Welsh (statutory), Urdu, Punjabi, Bengali, Somali, Gujarati as minimum
WCAG 2.2 AA — with additional cognitive accessibility for dementia and LD users
Proxy access — designed for family members and advocates acting on behalf of the person
Mobile-first — many informal carers access services on mobile while caring
Plain English — FK Grade 6 or below; jargon-free (no "wellbeing outcomes" unexplained)
No login required for rights guidance — consent-first for personal data
GDS service standard — all 14 points assessed, with emphasis on #1 (user need) and #5 (iterations)
Culturally sensitive — family structures and attitudes to care vary significantly
Rural accessibility — rural populations face compounded access barriers; location-aware
Always routes to a human — including out-of-hours emergency care contacts
Signposts CQC ratings clearly — helps people make informed choices about providers
Co-designed with communities — people with lived experience involved from alpha

Recommended build order

  1. 1

    care_explain_rights

    Simplest to build, highest foundational value. GOV.UK Care Act statutory guidance via Content API. Build first — rights knowledge is the prerequisite for everything else.

  2. 2

    care_find_services

    CQC Provider API is free, excellent and well-documented. Can be built independently of the rights/eligibility logic. Shows tangible results quickly.

  3. 3

    care_check_eligibility

    Requires implementing Care Act wellbeing outcome domains as structured TypeScript data. Complex but high value — many people are eligible and don't know it.

  4. 4

    care_navigate_assessment

    Builds on rights and eligibility tools. Content primarily from GOV.UK guidance plus synthesis of what assessment conversations involve.

  5. 5

    care_explore_funding

    NHS CHC pathway is complex but highest financial value. Requires encoding CHC decision support tool criteria and funding rules.

  6. 6

    care_find_advocacy

    Requires IMCA/Care Act advocate directory. Build last but ensure a hardcoded fallback to Age UK (0800 678 1602) is present from day one.

Questions for the lab

  2. Is there a model for how local authority care data will eventually be accessible via NDL, given the 152-council fragmentation problem?
  3. CQC has an excellent public API — are there other public sector bodies in the social care space with comparable APIs we should be aware of?
  4. The NHS CHC pathway is systematically underused — is there existing DH/NHSE data on how many eligible people don't receive it?
  5. Who should co-design this with us? Recommend: Disability Rights UK, Age UK, Carers UK, ADASS, and at least two local authority social care teams.
