Legal Guidance MCP

Scope & Architecture · NDL Kickstarter Domain 4

Draft v0.1 Cross-Gov AI Engineering Lab June 2026 For review and comment

Purpose. This document scopes an MCP server enabling AI assistants to provide citizens — particularly those who cannot afford legal advice — with accurate, source-cited guidance on their legal rights, time limits, and how to access free help. Originally scoped by DSIT for SMEs using authoritative legal records; this scope extends the citizen-facing application to the highest-impact use cases: housing disrepair, benefit appeals, and employment disputes.

Important legal boundary. This MCP provides legal information, not legal advice. Output must clearly distinguish between "here is what the law says" and "here is what you should do in your specific situation." Every flow must offer a route to qualified legal help. This boundary must be designed in, not bolted on.

Problem statement

The UK has one of the largest legal advice deserts in the developed world. Legal Aid was cut by 35% between 2010 and 2020. Citizens Advice handles 2.7 million cases annually with insufficient capacity. Benefit claimants face mandatory reconsideration deadlines they don't know exist. Private tenants live in dangerous conditions they don't know they can challenge. Employment tribunal applications have 3-month time limits that expire before people know they have a case.

The NDL Kickstarter on AI-powered legal guidance uses authoritative legal records to enable AI guidance. This MCP targets the citizen layer: where information asymmetry between government/employers/landlords and citizens is greatest, and where a time limit running down can close off remedy permanently.

MCP tool specifications

legal_explain_rightsBuild first
Returns specific statutory rights for a given situation — plain English, with the legal source cited. This is the foundational tool. Uses legislation.gov.uk API and GOV.UK guidance. Every right must be sourced so users can verify independently.
interface LegalExplainRightsInput { situation_type: string; // "housing_disrepair"|"benefit_appeal"|"holiday_pay"|"unfair_dismissal" specific_context?: string; // free text: e.g. "landlord hasn't repaired damp for 8 months" reading_level?: 'simple' | 'standard'; language?: string; } interface LegalExplainRightsOutput { rights: LegalRight[]; legal_sources: LegalSource[]; // { act_name, section, legislation_url } — independently verifiable what_this_means: string; // plain English summary time_limits: TimeLimit[]; // CRITICAL: always surface time limits next_steps: string[]; disclaimer: string; // "This is legal information, not legal advice." }
legal_check_deadlines
The most time-critical tool in the suite. Many legal remedies are permanently lost because citizens don't know the time limit is running. Returns all applicable deadlines for a situation with days remaining calculated from the triggering event date.
interface LegalCheckDeadlinesInput { situation_type: string; trigger_date: string; // ISO date — e.g. date of refusal letter, date of dismissal council_area?: string; // some limits vary by council/jurisdiction } interface LegalCheckDeadlinesOutput { deadlines: Deadline[]; most_urgent: Deadline; // the one expiring soonest days_remaining: number; urgency_level: 'critical' | 'urgent' | 'standard'; if_missed: string; // what happens if deadline passes exceptions: string[]; // grounds for extension (e.g. mental health, disability) }
legal_assess_situation
Identifies the legal issue type from a free text description, maps it to relevant legal frameworks, and returns the most likely remedy. Entry point for users who don't know what type of problem they have — "is this even something I can do anything about?"
interface LegalAssessSituationInput { description: string; // free text description of the problem location: string; // postcode — jurisdiction matters } interface LegalAssessSituationOutput { legal_area: string; // "housing"|"employment"|"benefits"|"consumer"|"debt" issue_type: string; // e.g. "housing_disrepair"|"PIP_appeal" has_legal_remedy: boolean; time_sensitive: boolean; recommended_next_tools: string[]; // which other tools to call next confidence: 'high' | 'medium' | 'low'; }
legal_find_free_help
Finds Legal Aid eligible solicitors, law centres, free legal clinics, and specialist advice services by postcode and issue type. Legal Aid eligibility check included. Always returns Citizens Advice as a fallback.
interface LegalFindFreeHelpInput { postcode: string; issue_type: string; legal_aid_eligible?: boolean; // if known language?: string; contact_pref?: 'phone' | 'online' | 'in_person'; } interface FreeHelpService { name: string; type: string; // "law_centre"|"legal_aid_solicitor"|"clinic"|"citizens_advice" specialisms: string[]; contact: ContactInfo; languages: string[]; legal_aid: boolean; wait_time_est?: string; }
legal_draft_correspondence
Helps draft legally correct letters and emails — to landlords (repair notices, s.11 notifications), employers (holiday pay claims, flexible working requests), DWP (mandatory reconsideration requests). Includes the correct legal references and tone.
interface LegalDraftCorrespondenceInput { correspondence_type: string; // "repair_notice"|"MR_request"|"holiday_pay_demand"|"ET1_preparation" recipient_type: string; // "landlord"|"DWP"|"employer"|"council" situation_detail: string; // relevant facts formality_level?: 'formal' | 'standard'; } interface LegalDraftCorrespondenceOutput { letter_text: string; legal_references: string[]; // acts/sections cited in the letter key_points_included: string[]; suggested_sending: string; // "recorded delivery and email" etc. next_steps: string; }
legal_guide_process
Step-by-step guide to a specific legal process — employment tribunal, First-tier Tribunal (benefits), county court housing claim, Housing Ombudsman complaint. Demystifies the process for first-time users who assume courts are only for lawyers.
interface LegalGuideProcessInput { process_type: string; // "employment_tribunal"|"FTT_benefits"|"county_court"|"housing_ombudsman" current_stage?: string; // if already in the process represented?: boolean; // has a solicitor or is self-representing } interface LegalGuideProcessOutput { stages: ProcessStage[]; forms_needed: FormRef[]; evidence_tips: string[]; hearing_prep: string; timelines: Timeline[]; costs: string; // usually free for claimants at ET/FTT }

Data architecture

Legislation.gov.uk API Excellent

Free, public, well-documented RESTful API returning full statutory text. The authoritative source for all rights cited. Core to the NDL legal kickstarter's use of authoritative legal records.

GOV.UK Content API

Plain English explanations of legal processes and rights — benefit appeals, employment rights, housing. Complements the statutory API.

Legal Aid Agency data

Registered Legal Aid providers by location and specialism (public data). Core for legal_find_free_help.

Tribunal/Ombudsman decisions

First-tier Tribunal and Housing Ombudsman publish decisions (public). Useful for calibrating success rates and what arguments work.

NDL authoritative legal records Phase 2+

The NDL kickstarter's specific innovation — a curated corpus of authoritative legal content. Will enable more precise and current legal guidance than scraping alone.

DEIA requirements

Multi-language — advice deserts hit BAME communities hardest; Welsh, Urdu, Somali, Polish, Arabic minimum
WCAG 2.2 AA — screen reader compatible; many users may be in distress
Mobile-first — many vulnerable users access on mobile only
Plain English — legal language must be translated, not reproduced; FK Grade 6
No login required for rights information — consent-first for correspondence drafting
Always cites source — every right includes the act and section for independent verification
Clearly labelled as legal information, not advice — prominent, consistent, not buried
Distress-aware — benefit appeals and housing crises are stressful; tone calibrated
Always surfaces free help — no flow ends without a free legal help option
Time limits always surfaced — never mentioned only when asked; proactively flagged
GDS service standard compliant — especially #2 (solve the whole problem)
Co-designed with communities — law centres, housing charities, welfare rights orgs from alpha

Recommended build order

  1. 1

    legal_explain_rights

    Simplest to build. Legislation.gov.uk API + GOV.UK Content API. Start with housing disrepair (Landlord and Tenant Act 1985 s.11) and employment rights (holiday pay — Working Time Regulations 1998). Both APIs are free and public.

  2. 2

    legal_check_deadlines

    Highest urgency. Build second because a missed deadline is irreversible. Requires encoding time limits as structured data — manageable but must be exhaustively verified.

  3. 3

    legal_assess_situation

    Entry-point tool that routes to others. Build after the core tools so it can route accurately.

  4. 4

    legal_find_free_help

    Requires Legal Aid Agency directory integration. Build after core information tools so there's something to refer users to help with.

  5. 5

    legal_draft_correspondence

    High value but requires careful quality control — letters must be legally accurate. Build after rights explanation tool is verified.

  6. 6

    legal_guide_process

    Most complex — requires encoding multiple tribunal and court processes. Build last but ensure a pointer to HMCTS guidance is present from day one.

Questions for the lab

  1. What is the NDL "authoritative legal records" corpus — is it legislation.gov.uk data, case law, or something else? Who owns it within government?
  2. What is the appropriate liability framework for AI-generated legal information? Has DSIT/MOJ taken a view on this?
  3. Time limits are the highest-risk element — what QA process should apply to ensure they're always accurate and current?
  4. Should legal_draft_correspondence output be reviewed by a qualified lawyer before going live, even in beta?
  5. Co-design recommendation: Law Centres Network, housing charities (Shelter, Crisis), welfare rights advisers, Citizens Advice — all have deep user insight here.
